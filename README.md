# The-Gaming-Room-ReadME
 Journal entry for Class Week 8
 

1. Briefly summarize The Gaming Room client and their software requirements. Who was the client? What type of software did they want you to design?

  - The Gaming Room wanted to expand Draw It or Lose It from an Android-only game into a web-based application that supports desktop and mobile users. The software needed to be scalable, secure, and accessible across multiple operating systems using a cloud-based server.

  
2. What did you do particularly well in developing this documentation?

  - I think I did a pretty darn good job comparing different operating platforms and explaining why a Linux-based cloud environment was the best choice. I connected my recommendations to the client's requirements instead of simply listing features.

  
3. What about the process of working through a design document did you find helpful when developing the code?

  - The design document helped organize the project before coding. Defining the requirements and architecture first made it easier to understand how the application should function and what technologies best fit the project.

  
4. If you could choose one part of your work on these documents to revise, what would you pick? How would you improve it?

  - I would expand the operating system architecture section by including more technical details about Linux, such as its kernel architecture and resource management, to better support my recommendation.

  
5. How did you interpret the user’s needs and implement them into your software design? Why is it so important to consider the user’s needs when designing?

  - I focused on the client's need for a secure, scalable, cross-platform application. Those requirements guided my recommendations for a Linux cloud server, browser-based clients, and distributed architecture. Meeting user needs is important because software is only successful if it solves the client's problems effectively.

  
6. How did you approach designing software? What techniques or strategies would you use in the future to analyze and design a similar software application?

  - I started by identifying the requirements and then compared different solutions before making recommendations. In future projects, I will continue using this structured approach by evaluating trade-offs, documenting the design early, and considering scalability, security, and maintainability throughout development.
